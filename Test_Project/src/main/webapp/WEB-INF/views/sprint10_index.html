Sprint 10

Upload de fichiers avec injection automatique

Fonctionnalités du Sprint 10

  • Classe FileUpload : Encapsule un fichier (nom + contenu en bytes)
  • Injection automatique : Les fichiers sont injectés comme paramètres de méthode
  • Upload multiple : Plusieurs fichiers dans un même formulaire
  • Sauvegarde facile : Méthode saveTo(directory) intégrée
  • Combinaison avec les sprints précédents : Upload + Objet, Upload + Map
  • Pas de code boilerplate : Le framework gère multipart/form-data

Test 1 : Upload simple

Objectif : Uploader un seul fichier avec une description.
Méthode : uploadSingle(FileUpload photo, String description)

@PostUrl("/sprint10/upload-single") public ModelView uploadSingle( FileUpload photo, String description ) { photo.saveTo("C:/uploads/"); }

Points importants

  • enctype="multipart/form-data" est OBLIGATOIRE dans le formulaire
  • Le name de l'input file doit correspondre au paramètre du contrôleur
  • Le chemin de sauvegarde par défaut est C:/uploads/ (modifiable dans le contrôleur)
  • La classe FileUpload contient : nom, contenu (bytes), type MIME, taille
  • Méthode saveTo(directory) : sauvegarde automatique sur le serveur
